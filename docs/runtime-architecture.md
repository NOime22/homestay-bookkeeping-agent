# Runtime Architecture / 运行时架构

> 这份文档描述一个**生产可行**的运行时是如何拼出来的——基于真实跑通的经验整理。具体宿主（Hermes / LangChain / Dify / 自研）的实现细节不在这里展开，请看你宿主自己的文档。

## 2. 关键设计决策

### 2.1 用"固定模板卡"，不要"即时拼装 JSON 卡"

**理由**：
- 模板卡有版本号，可以稳定回滚
- 卡片美化由飞书可视化编辑器完成，代码侧只管 variables
- 客户端兼容性远好于自建 card JSON

**代价**：
- 模板字段一旦定型，加字段要发新版本
- 需要在卡片可视化编辑器和代码之间维护变量对照

### 2.2 下拉只更新 draft，不写账

**理由**：
- 每次下拉变化都写账 = 海量 dirty write
- 用户思考过程中改改换换是常态
- 写账放在最后一步"确认记账"才稳

### 2.3 金额/日期走"文字修改"，不走卡内输入

**理由**：见 [`../LESSONS_LEARNED.md`](../LESSONS_LEARNED.md) 第 1.3 节。`input` / `date_picker` 在 CardKit 模板卡里不可靠。

### 2.4 证据图上云盘 + 链接列，不上附件列

**理由**：见 [`../LESSONS_LEARNED.md`](../LESSONS_LEARNED.md) 第 6.1 节。

## 3. 状态机

每张活跃的 draft 至少要持久化以下状态：

```
candidate_ready
    ↓
card_sent ───────► 用户点下拉 ──► card_edited（可重复）
    │                                │
    │                                ▼
    └──► 用户点"有文字修改" ──► awaiting_text_reply
                                     │
                                     ▼
                              用户回复文字 ──► card_edited（刷新卡）
                                     │
                                     ▼
                              用户点"确认记账" ──► confirmed ──► written
                                                                  │
                                                                  ▼
                                                                 ✅
```

每个 draft 至少存：

- `candidate_id`
- `chat_id`
- `actor_id` / `sender_id`
- `source_message_id`
- `target_table` / `ledger`
- `edited_fields` 快照
- `text_edit_prompt_message_id`
- `awaiting_text_reply`
- `updated_at`

## 4. 宿主无关原则

这个 Agent 不依赖任何具体框架。任何宿主只要能：

1. 收飞书事件、发飞书消息（用 SDK 或 OpenAPI 都行）
2. 跑一个 vision-capable LLM
3. 持久化 draft 状态（内存、SQLite、Redis 均可）
4. 调飞书云盘 API 上传文件
5. 调飞书多维表格 API 增删改查

……就能跑这个 Agent。

实现细节见你的宿主自己的 adapter 层。本仓库里的 [`adapters/feishu.md`](../adapters/feishu.md) 给了一份通用建议。

## 5. 必跑的回归项

每次升级宿主或模型，至少要回归这些路径，否则容易隐性退化：

- [ ] 启动消息只切角色不发卡
- [ ] 截图判型走主模型原生看图（不是 OCR 降级）
- [ ] 截图 + 文本混合时，截图优先判型
- [ ] 卡片实际渲染（不仅 API 200）
- [ ] 下拉点击后 loading 能停（card + toast 都返回）
- [ ] "有文字修改"路径能完整跑回刷新卡
- [ ] 最终写账：表格里看到新行
- [ ] 证据图 URL 能在表格里打开
